# No Generic Web — v3.2

**Build immediately.** The request is the confirmation.

---

## What This Solves

AI-generated sites converge to the same output: centered hero, Inter font, blue/gray palette, 3-column card grid, "Get Started" CTA. They're identical because they're generated from the same implicit template — not from what the brand actually is.

This skill replaces template thinking with decision thinking. Every output must make choices that could only belong to this brand, this goal, this audience.

---

## Layer 1 — Pre-Design: Three Questions First

Answer before any visual decision. These govern everything else.

**Business intent:**
- What single action should this page produce? (buy / sign up / book / contact / explore / trust)
- What must the user understand in the first 5 seconds? (who it's for, what it does, why it matters)
- What objection arrives with the user? (too expensive / too complex / not for me / I don't trust this yet)

**User journey:**
- What does the user already know when they land? (cold traffic vs. warm lead changes everything)
- What do they need to believe to convert? That belief gap is the page's real job.
- What emotional state produces conversion? Confidence? Curiosity? Reassurance?

**Page narrative:**
Pages are arguments, not collections of sections. Every section serves one of four narrative roles:

1. **Claim** — What this is and why it matters (hero)
2. **Evidence** — How it works, what you get (features, process)
3. **Trust** — Why believe it (social proof, credibility)
4. **Resolution** — The conversion moment (CTA with low friction)

If a section doesn't serve one of these roles, cut it.

---

## Layer 2 — Brand DNA

Extract from everything given: name, description, industry, audience, tone, existing assets, vibe words in the message.

**Four commitments before writing code:**

1. **Typographic voice** — Not "which font" but what feeling: authoritative / warm / precise / literary / raw / playful / clinical
2. **Color personality** — Not just hues but emotional register: calm / tense / vibrant / subdued / warm / cool / austere / electric
3. **Spatial energy** — How does the page breathe? Compressed and urgent, or spacious and deliberate?
4. **Compositional stance** — Does the layout assert itself, or serve the content quietly?

**Category signals → honest translations (no clichés):**

| Category | Avoid | Do instead |
|---|---|---|
| Luxury / premium | Black + gold, marble, ornate serif | Restraint, earned whitespace, confidence that doesn't perform |
| Playful / consumer | Cartoonish, random, rounded corners everywhere | Warm, emotionally open, intentional energy — not simplified |
| Technical / dev | Blue glow, generic startup dark | Precision spacing, information hierarchy, credibility through clarity |
| Organic / craft | Fake textures, stock farm photos | Specific materiality, honest warmth, imperfection that feels real |
| Bold / agency | Chaos, everything oversized | Confident decisions, clear hierarchy, visual aggression with control |
| Wellness | Generic green, spa cliché | Specific calm — air, light, quiet — not performed serenity |
| Financial / trust | Default navy + white + gray | Authority through specificity. Trust via clarity, not just formality |

**Brand check:** Could this palette + typographic voice + spatial energy belong to a different brand? If yes — not specific enough.

If user provides brand assets — they are law. Extend them, never replace them.

## Layer 3 — Mode System

### A. Base Mode — choose one

The base mode sets the fundamental logic of the page. It determines what the design prioritizes and how it resolves conflicts between brand expression and function.

---

**Brand-First**
*Portfolios, agencies, creative studios, cultural organizations, brand campaigns.*

The site must feel like the brand itself — identity is the primary deliverable. Impression comes before immediate conversion. Layout, typography, and color carry meaning. A visitor should feel something before they understand anything.

Use when: the product is the experience, the portfolio is the pitch, or the brand's character is the differentiator.

Avoid: conversion pressure before conviction is built. Functional patterns that undermine the brand feeling.

---

**Conversion-First**
*Landing pages, lead gen, local business, service businesses, single-offer pages.*

One action. Everything else is support. Clarity beats cleverness — the value proposition must be legible immediately. The CTA's label, position, and visual weight are the most important design decisions. Social proof and objection resolution must appear before the ask.

Visual personality still matters — generic kills trust — but it cannot slow comprehension.

Use when: the goal is measurable and singular, and user decision-making is the bottleneck.

Avoid: clever layouts that hide the CTA. Branding that competes with the conversion path.

---

**Product / SaaS**
*Apps, platforms, B2B tools, developer products, multi-feature software.*

Users arrive skeptical. The job: comprehension → credibility → confidence → action. Section order matters: value prop → how it works → proof → pricing → CTA. Users scan first, read second — each section must communicate at a glance.

Use when: the product's complexity needs to be made simple, or the value is functional and needs demonstration.

Avoid: aesthetic-first decisions that sacrifice comprehension. Features that aren't shown in use.

---

**Editorial**
*Blogs, magazines, case studies, manifestos, long-form journalism, deep content.*

The reading experience is sacred. Typography is the primary design tool — it must be exceptional. Hierarchy through type scale, weight, and spacing, not layout tricks. Visual moments punctuate — they don't compete with the text.

Use when: the content is the product and reading quality is the experience.

Avoid: visual distractions that interrupt reading flow. Complex layouts that fight the content.

---

**Experimental**
*Art projects, campaign sites, portfolio showpieces, unconventional brand experiences.*

Conventions are starting points, not limits. Complexity is earned, not assumed. Controlled audacity — not noise. Every unusual choice must serve the experience.

Use when: the project explicitly calls for it or the brief rewards creative risk.

Avoid: defaulting here when another mode fits. Complexity that confuses rather than provokes.

---

### B. Submode — activate when relevant

Submodes refine the character within the base mode. They're not visual styles — they're a set of calibrated adjustments to tone, composition, density, motion, and UX behavior. A submode is invisible if you're doing it right: felt, not labeled.

Activate a submode when: the user signals it explicitly ("luxury", "bold", "minimal"), or when the brand's character clearly demands it.

---

**Luxury**

The feeling of confidence that doesn't need to perform. Luxury is restraint with conviction — not decoration, not emptiness, not default black/gold.

*Adjusts:*
- Restraint is the primary tool. Every element earns its presence. When in doubt, remove.
- Space is the luxury signal — not gold, not marble, not serif by default. Generous, considered spacing.
- Typography: one exceptional typeface used with mastery, or two in quiet contrast. Can be sans-serif. Luxury is in how type is used, not which category it belongs to.
- Color: must be earned and specific. Could be warm stone + deep ink, aged ivory + graphite, chalk + bronze. Never defaults (black/gold, white/serif).
- Motion: long, slow easing. No bounce or spring. Deliberate. Arrivals and departures are composed.
- Interaction: invisible until needed. Precision hover states, not theatrical reveals.
- Density: one focal point per viewport. Elements competing for attention is a failure.
- Copy: fewer words, more weight. Authority without aggression.

*Avoid:* Black + gold as "luxury shorthand". Marble textures used decoratively. Empty pages that confuse restraint with laziness. Theatrical scroll effects. Ornate typography for the sake of premium signal.

---

**Minimal**

Restraint as personality — not the absence of it. Minimal is an argument made with as little as possible. It requires more discipline than any other mode, not less.

*Adjusts:*
- Every element must justify presence. Removing it should cost something.
- Whitespace is active — it creates focus, rhythm, and weight. Not padding.
- Typography carries the visual load. If it's not exceptional, minimal fails entirely.
- Color: 2–3 maximum, specific. "Neutral" is not a strategy. Earn the palette.
- Motion: near-absent. Only when it aids comprehension or creates meaningful rhythm.
- Structure: disciplined grid. Breakouts only when they earn significant visual focus.

*Avoid:* Blank ≠ minimal. Empty without intention is unfinished. Minimalism without a point of view is just incomplete design.

---

**Bold**

Directness as a design value. Bold is a compositional and editorial posture — not a visual intensity setting. A bold site says what it means immediately and without apology. Everything visual serves that directness.

**Bold vs. intensity=Expressive:** Intensity controls *how loud* the expression is. Bold controls *what the page is saying*. You can have Bold+Restrained — understated directness, compressed copy, single unambiguous claim, quiet but immovable. Bold+Expressive is the declarative statement pushed to its visual limit. The confusion is common: Bold without hierarchy is just volume. Bold is about stance, not decibels.

*Adjusts:*
- **Compositional logic:** The layout makes a statement. Not just interesting — *declarative*. Structure asserts, doesn't just arrange. The page "says" something with how it's built.
- **Typographic logic:** The most important thing is always the biggest thing. Hierarchy is radical, not moderate. Headlines are declarations, not descriptions. Body copy is proportionally smaller — contrast earns readability.
- **Editorial logic:** Less copy, stronger claims. No hedging, no qualifiers. One clear thing per section. The brand says what other brands won't.
- **Color logic:** Palette creates pressure — not just personality. Two strong tones in deliberate tension. Color is a statement, not a backdrop.
- **Motion logic:** When it moves, it arrives decisively. No gentle fade-ins. Cuts, snaps, or precision eases that feel purposeful. At Restrained intensity: no entrance animation — the static composition does the work.
- **Interaction logic:** Hover states respond with confidence. They don't suggest — they react. The UI has posture.
- **Emphasis budget:** Bold demands a strict single primary zone per viewport. Multiple competing bold elements cancel each other. One thing is bold. Everything else yields to it.

*Avoid:* High volume without hierarchy is noise wearing boldness as a costume. If everything is bold, nothing is. The dominant element must earn its dominance — it must be the most important thing, not just the biggest thing. Bold ≠ expressive intensity applied everywhere.

---

**Tech**

Credibility through precision and intelligence — not blue glows. Tech is a UX stance: information over impression.

*Adjusts:*
- Spacing: mathematical, grid-aligned. Precision signals credibility.
- Color: often dark ground. Accent = information signal, not decoration. Never decorative glow.
- Monospace: use intentionally for code, data, or specific credibility moments. Not globally.
- Motion: functional — reveals that aid comprehension. Nothing ambient or decorative.
- Density: moderate to high. Technical audiences scan efficiently.
- Hierarchy: serves comprehension above aesthetics. Type weight and scale signal importance.

*Avoid:* Blue glow, generic "startup dark", neon accents as decoration, dark mode as visual style rather than functional choice.

---

**Corporate**

Trustworthy and specific — without being lifeless. Corporate is not boring. The enemy is lazy generic, not professional seriousness. A corporate site can have strong visual identity.

*Adjusts:*
- Structure signals reliability. Clarity and predictability are features, not limitations.
- Color: specific and earned. Default navy + gray + white is lazy — not safe.
- Typography: authoritative and legible. Can have personality within those constraints.
- Motion: measured. Respects the user's time. No transitions that make people wait.
- Density: higher density is appropriate — professional audiences are information-comfortable.
- Hierarchy: functional. The user always knows where they are and what to do next.

*Avoid:* Generic = amateur, not safe. Serious tone and visual strength are not opposites.

---

**Playful**

Emotionally open and inviting — not childish or patronizing. Playful means warm, accessible, and specific. It can be sophisticated.

*Adjusts:*
- Color: saturated, warm, energetic — but purposeful. Not random.
- Motion: expressive and springy — in service of feedback and delight, not performance.
- Typography: friendly, approachable — but legible. Display type with genuine personality.
- Layout: relaxed rhythm, informal — but intentional. Asymmetry with purpose.
- Interaction: curiosity rewarded. Hover states that surprise in a delightful way, not a confusing one.
- Copy: warm, conversational, specific. Reads like a person with a personality.

*Avoid:* Rounded corners everywhere, emoji as design elements, oversimplification. Playful ≠ simplified for a perceived lower-capability audience.

### C. Intensity — operational modifier

Sets how assertively the chosen mode + submode expresses itself. Modifies six design variables simultaneously. Default: **Balanced.**

|  | Restrained | Balanced | Expressive |
|---|---|---|---|
| **Scale contrast** | Subtle — headline 2–3× body size | Moderate — 3–5× | Dramatic — 5× or more; type functions as visual element in at least one section |
| **Color pressure** | Low saturation, minimal accent, palette recedes into background | One color with personality, clear hierarchy signal | Strong palette tension — color makes a statement, not just a decision |
| **Compositional gestures** | 1 intentional structural decision per page | 2–3 across the page | 4+ — layout direction commits fully to its logic |
| **Motion** | Entrance fades only, ≤300ms, no scroll animation | Entrance reveals + purposeful hover states + 1 scroll-linked behavior | Scroll-driven animation, ambient motion where earned, choreographed sequences |
| **Ornament / atmosphere** | None unless it carries specific brand meaning | Sparingly when it extends character | Welcome when it reinforces the design language |
| **Copy register** | Understated — precision over volume, no hyperbole | Confident and specific, some voice | Voice-forward, opinionated, memorable — says things other brands wouldn't |

**Override cases:**
- Luxury + Restrained: understatement is the argument. The brand earns trust through what it withholds.
- Bold + Expressive: scale contrast and color pressure pushed to their logical limits. The layout makes a declaration.
- Minimal + Restrained: reduction taken furthest — every remaining element is load-bearing.
- Playful + Expressive: color, motion, and copy all lean into warmth simultaneously without canceling each other.
- Corporate + Restrained: professionalism through discipline. No decoration, but specific and strong.
- Tech + Balanced: precision without coldness — technical credibility doesn't require austerity.

---

### D. Thirdmode — activate for scroll-driven spatial narratives

Thirdmode is a structural layer, not a visual style. It changes how the page is organized, paced, and sequenced. Submode modifies character. Thirdmode modifies architecture.

Only one thirdmode: **3D Scroll Mode**

Thirdmode does not replace base mode or submode. Those layers continue governing character, conversion logic, and tone. Thirdmode overrides layout and pacing architecture within that context.

---

**3D Scroll Mode**

The page is a scroll-driven narrative. Sections are chapters. The user's position in the scroll is their position in a story or space. Motion is tied to scroll progress, not decoration.

*Use when:* Architecture, automotive, premium physical products, real estate, immersive brand launches, product storytelling with spatial or material character.

*Do not use when:* Fast-conversion pages, information-dense sites, dashboards, utility tools, SaaS with complex feature comparison, anything where scroll pacing slows a decision.

**Two variants:**

**Object-led** — A protagonist object (car, watch, device, packaging, industrial product) guides the experience. Scroll reveals, transforms, or narrates the object. Every chapter returns to the object as the organizing center. Motion is tied to object state. Object in foreground, context behind.

**Space-led** — The experience moves through a place: architecture, interiors, hospitality, real estate, material storytelling. Scroll = traversal. Each chapter is a spatial moment — a room, a threshold, a material shift. Motion is camera-like: progressive depth, environmental transitions, light and surface changes. Not "object rotating." Think: arriving, passing through, approaching.

*What 3D Scroll Mode modifies:*

| Variable | Behavior |
|---|---|
| **Narrative structure** | Chapter-based. Each scroll zone = one narrative beat, one function. No block-grid thinking. |
| **Layout** | Sticky sections allowed. Layered transitions expected. Scroll-linked reveals as structure, not decoration. |
| **Content density** | Compressed. Max 2–3 elements per chapter. Copy serves the experience — doesn't compete. |
| **Motion** | Tied to scroll progress. Transitions support narrative progression. No ambient decorative loops. |
| **UX obligations** | Scroll state must feel legible at all times. CTA path must remain findable. Mobile = simplified, not degraded: redesign the pacing for touch, don't just stack it. |
| **Technical floor** | Faux-3D / layered illusion is acceptable. True 3D (Three.js / WebGL) only if content and budget justify. Never assume heavy implementation. |

*Technical implementation levels — use the lightest that works:*
1. Layered depth illusion — CSS z-index, scale, opacity, blur
2. Parallax depth — foreground/background at different scroll rates
3. Scroll-linked transforms — GSAP ScrollTrigger or CSS `@scroll-timeline`
4. Image-sequence storytelling — frame-by-frame tied to scroll position
5. CSS/GSAP spatial motion — 3D transforms without a 3D engine
6. True 3D — Three.js / WebGL only when content earns the cost

*Anti-patterns specific to 3D Scroll Mode:*
- Spatial experience where the proposition stays unclear — spectacle without substance
- Motion-heavy intro with weak information payoff — earns no trust, just delays
- Object-led logic applied to a space-led project (and vice versa) — wrong organizing center
- No conversion logic in the scroll narrative — immersive with nowhere to go
- Mobile degradation treated as acceptable — touch pacing needs its own design
- More than 2 layers moving simultaneously — orientation collapses
- Cinematic pacing that becomes slow or frustrating — users scroll to progress, not to wait
- Technical ambition that breaks performance — 3D scroll that loads slowly is not premium
- Three.js for its own sake — if removing the spatial layer makes the content clearer, remove it

---

### Layer integration

Priority when layers conflict: **business clarity → UX legibility → narrative coherence → style.**

| Layer | Governs | Never overrides |
|---|---|---|
| Base mode | Fundamental page logic — what it prioritizes | Nothing. It defines the contract. |
| Submode | Compositional character and posture | Base mode constraints |
| Thirdmode | Page architecture and scroll sequencing | Base mode + submode logic still applies inside chapters |
| Intensity | Expression level of mode + submode | Any conflict: lower intensity wins |

**Valid combinations:**
- Brand-First + Luxury + 3D Scroll (Space-led) → architecture or hospitality brand where identity and space are one. Restrained is the natural intensity — the composition does the work.
- Brand-First + Luxury + 3D Scroll (Space-led) + Restrained → maximum compositional discipline. Every chapter earns its moment.
- Brand-First + Bold + 3D Scroll (Object-led) + Balanced → product launch with strong visual confidence. Object is the argument.
- Experimental + Bold + 3D Scroll (Object-led) + Expressive → high-commitment experience. Valid only if content earns it. Risk of spectacle over substance — monitor.

**Invalid or high-risk combinations:**
- Conversion-First + 3D Scroll → structurally incompatible. Immersive pacing conflicts with decision speed. If conversion is singular and measurable, 3D scroll hurts it.
- Editorial + 3D Scroll → reading is the experience. Spatial motion competes with text. Don't.
- Product/SaaS + 3D Scroll → rarely justified. SaaS comprehension requires efficient scanning. Only if the product's physical form makes spatial storytelling genuinely useful (hardware product, physical interface).
- Corporate + Expressive → misaligned. Professional audiences read high intensity as unfocused.
- Minimal + Expressive → self-defeating. Minimal argues through reduction. Expressive intensity undermines the argument.

## Layer 4 — UI/UX Judgment

Visual quality and UX quality are not the same thing. Apply this layer before outputting. These are decision rules, not principles.

### Above-the-Fold Stack

What the user must resolve before they scroll. Priority order by base mode:

- **Conversion-First:** (1) Who this is for + what it does → (2) Primary credibility signal → (3) CTA
- **Product/SaaS:** (1) Value proposition + audience signal → (2) Key differentiator → (3) Low-friction entry (free trial, demo, pricing)
- **Brand-First:** (1) Emotional impression → (2) What this actually is → (3) Entry point (nav or first action)
- **Editorial:** (1) What this is about + who wrote it → (2) First piece of content or reading entry

If the above-fold fails its stack — the page fails, regardless of how good everything below is.

### Emphasis Budget

Each viewport has one attention budget. Spend it deliberately:
- **1 primary zone** per viewport — maximum visual weight. Everything else defers to it.
- **1–2 supporting zones** — clearly secondary by scale, contrast, position, or isolation.
- **Atmospheric layer** — background elements that set tone without competing.

If 3+ elements feel equally important within the same viewport: reduce one. Elements competing for attention produce no attention.

### Trust Signal Placement

Trust signals earn their position relative to what they're asked to support:
- **Cold traffic (ads, search):** trust signal in the hero or immediately below. Don't make them scroll for it.
- **Warm traffic (referral, direct):** trust can follow the evidence section.
- **High-stakes decisions (high price, long commitment, sensitive data):** mandatory trust before the CTA.
- **Low-stakes decisions (free trial, no-card signup):** CTA can lead, trust follows.

### Density by Context

How many distinct content elements per section — calibrated to mode and audience:

| Context | Max visible elements per section |
|---|---|
| Conversion-First, above fold | 3 (headline, value signal, CTA) |
| Product/SaaS, feature sections | 4–6 with clear hierarchy |
| Brand-First + Luxury | 1–2 focal elements; scroll creates progression |
| Editorial body | Text block + 1 optional visual accent |

Overwhelmed = reduce. Unconvincing = increase. Don't default to the middle without deciding.

### Section Pacing

Pages need alternating rhythm. Dense sections need release:

Typical rhythm: **atmospheric** → **dense evidence** → **rest beat** → **moderate features** → **dense trust** → **focused CTA**

Rules:
- Two consecutive dense sections: acceptable if they serve different narrative roles
- Three consecutive dense sections: insert a rest beat (full-width image, single stat with space, a quote, generous empty section)
- Every page needs at least one pause moment — a section where the user absorbs rather than scans

### Conversion Path Clarity

The user must always know what to do next:
- One primary CTA. Secondary action must be visually subordinate (lower weight, smaller, less prominent color).
- Marketing site navigation: 4–5 items max. More items = more escape routes from the conversion path.
- CTA must be visible without horizontal scrolling at every breakpoint.

### Form Friction Rules

Every field added reduces conversion probability:
- Above-fold or first-touch forms: **max 2 fields** (email + submit is ideal)
- Longer forms: **max 5 fields** — each field needs explicit justification
- Never ask for information before you need it. Name comes after signup. Company comes during onboarding. Phone comes only when the business model requires it.
- Required fields only. If it's optional, don't show it at the first touchpoint.

### Mobile Hierarchy

Mobile isn't desktop scaled down. It has its own priority logic:
- Above-fold on mobile: headline + one value signal + CTA. Nothing else competing.
- Primary CTA must be reachable without scrolling on 375px, or fixed in the nav bar.
- Thumb zone (lower 60% of screen): primary interactive elements should live here.
- Type: headline min `2rem` on 375px viewport. Body min `1rem`. Never smaller.
- Navigation: collapsed by default. Accessible via minimal, standard pattern.

### When Visual Design Breaks UX

Signals that design execution is overriding judgment:
- The hero is beautiful but fails the 5-second test — unclear what it is or who it's for
- The layout is interesting but the CTA is hard to find
- The motion is impressive but delays content comprehension
- Section transitions are creative but the narrative thread is lost between them
- The branding is specific but the value proposition is generic
- The design has personality but the hierarchy doesn't guide the eye
- The site looks polished but the mobile CTA requires scrolling 3 screens

When any of these are true: fix the UX failure first. Visual polish on a broken experience is misdirected effort.

## Layer 5 — Visual Decisions

### Typography

Character comes from intention, not quantity of fonts. One typeface used with mastery outperforms two used without thought.

- **Contrast approach:** two typefaces with clear role separation (display + body), OR one typeface with dramatic scale, weight, and tracking variation.
- **Scale:** headlines must command. Don't whisper. `clamp(2.5rem, 5vw, 6rem)` is a starting point, not a ceiling.
- **Weight and tracking:** vary with purpose. All-caps tight tracking for labels. Loose tracking for large display text. Normal tracking for body.
- **Line height:** body 1.5–1.7. Display 0.9–1.1. Never browser defaults.
- **Typography must communicate hierarchy without color.** Remove all color — if hierarchy collapses, the type system is broken.

### Color

- Palette must have emotional specificity, not just visual acceptability.
- At least one color decision creates personality or tension. "Clean" is not a color decision.
- Backgrounds: off-whites, warm creams, cool grays, deep navies, warm darks — all valid. White and black are choices, not defaults.
- Color as semantic signal: use it consistently to mark actions (CTA), information (text), structure (background). Don't mix semantic roles.

### Layout

Choose a compositional direction before writing code. Commit to it.

| Direction | Logic |
|---|---|
| **Conversion Column** | Single path, no distractions. Every element serves the CTA. |
| **Split Asymmetric** | Visual tension, identity assertion, authority. Brand-First. |
| **Vertical Storytelling** | Narrative builds as user scrolls. Product launches, case studies. |
| **Editorial Column** | Reading-optimized. Long-form, manifesto, interview. |
| **Modular Grid** | Multi-feature, comparison, showcase. Broken by at least one intentional variation. |
| **Single Statement** | One message, maximum space. Minimal brands. Requires exceptional typography. |

Asymmetry is a tool, not a requirement. A precise grid with exceptional type can be more distinctive than an asymmetric layout with generic content.

### Motion

Motion budget by base mode:

| Mode | Budget |
|---|---|
| Conversion-First | Minimal. Subtle entrance reveals. Purposeful hover. Nothing delays the CTA. |
| Product/SaaS | Functional. Reveals that aid comprehension. Micro-interactions that confirm state. |
| Brand-First | Intentional. Reflects brand character. Scale with submode + intensity. |
| Editorial | Near-still. Gentle fade-ins if anything. Reading is the experience. |
| Experimental | Motion as design material. Choreographed, continuous where it earns its place. |

Every interactive element gets a custom state. Custom means intentional — not complex. A precise underline grow is custom. `opacity: 0.8` is default.

Easing: `cubic-bezier(0.16, 1, 0.3, 1)` for entrances. `cubic-bezier(0.4, 0, 1, 1)` for exits. Not `ease` or `linear` across the board.

## Anti-Patterns

Two categories: obvious defaults (easy to catch) and sophisticated failures (harder — they often look like good design).

### Obvious Defaults
These are defaults, not absolute bans. Using them requires a clear reason:

**Layout:** centered hero (H1 + subtitle + CTA stack) / three equal-weight feature cards after the hero / every section alternating left-right / footer with four identical columns

**Visual:** blue/gray/white palette (`#3B82F6` + white) / blue-to-purple decorative gradient / `Inter` as the only font with no intent / rounded rectangles everywhere without variation / decorative icons that carry no information

**UX:** CTA before trust has been established / nav with 7+ competing links on a marketing site / forms asking for more than needed above the fold / identical density and rhythm across all sections

**Copy:** "Welcome to [Product]" / "The best [X] for [Y]" / "Powerful features. Simple design." / "Get started today." / "Learn More" / any headline that works on a competitor's site

### Sophisticated Failures
These are harder to catch because they often look good. They pass visual inspection but fail quality inspection:

- **Distinctive visuals, generic proposition.** The brand looks specific — custom typeface, strong palette, original photography — but the headline could belong to any competitor. Identity without offer is decoration.
- **Atmospheric design with weak proposition clarity.** Beautiful sections that fail the 5-second test. Impressive to scroll, unclear to understand. Impression over communication is an error when the user hasn't yet decided to care.
- **Section variety without narrative logic.** Different layouts, colors, and treatments per section — but no through-line argument. Feels like a design exercise, not a page with a job.
- **Branding that competes with conversion.** Brand expression so dominant it obscures what the user is supposed to do. Identity should build the case — not replace it.
- **Fake originality through decorative disruption.** Unusual angles, overlapping elements, irregular grids — but the disruption doesn't serve the content. Visual restlessness is not the same as visual intention.
- **Premium styling with hidden usability failures.** Looks luxury, but the CTA is hard to find, the nav is ambiguous, or the hierarchy doesn't guide. Elegance doesn't excuse confusion.
- **Excessive contrast without hierarchy.** When everything is high-contrast, nothing has contrast. Visual emphasis requires a hierarchy of emphasis.
- **Specificity in visuals, genericness in the offer.** The design makes a specific impression but the copy doesn't. "We build remarkable digital experiences" is as generic on a beautiful site as on a plain one.
- **Motion that delays comprehension.** Complex entrance choreography or continuous ambient animation that makes the user wait to understand what the page is about. Motion earns its cost by aiding understanding or creating appropriate feeling — not by performing.

## CTA Intelligence

Every CTA must derive from what this specific business actually does.

**Test:** Could this exact CTA appear on a different type of business? If yes, rewrite it.

- "Get Started" on luxury brand → "Discover the collection"
- "Subscribe" on niche community → "Join the circle"
- "Contact Us" on law firm → "Book a private consultation"
- "Learn More" (anywhere) → name what they'll learn or see
- "Book a demo" on a B2C app → probably the wrong ask entirely

CTA label, position, visual weight, and surrounding context work as a system — not as an isolated element.

---

## Copy

Real content. No placeholders, no lorem ipsum, no template language.

- **Headlines:** specific and evocative. Could not work on a competitor's site.
- **Subheadlines:** give real information or genuine intrigue. Don't restate the headline.
- **Body:** sounds like a person who understands the product and its audience.
- **CTAs:** written from what the user wants to do, not what the business wants to say.
- **Social proof:** specific — company names, real outcomes, numbers that mean something.

## Technical Baseline

- Self-contained HTML/CSS/JS unless framework specified
- CSS custom properties for all design tokens (colors, type scale, spacing)
- Responsive: 375px / 768px / 1280px+. Mobile must have intentional layout decisions, not just stacked columns.
- Prefer CSS for simple motion; JS for scroll-linked or complex interaction-driven behavior
- `loading="lazy"` on below-fold images. Meaningful `alt` text always.
- Real imagery (Unsplash/Pexels) when the project represents a physical product, place, or person. No posed generic stock.
- Scope nav CSS to specific selectors (`nav#main`, `.site-nav`) — bare `nav` selects footer navs too
- Never native `<select>` in custom-themed designs — OS renders it white regardless of CSS
- `@media (pointer: fine)` guard on hover-only effects — disable on touch devices
- Flex nav: `min-width: 0` on shrinkable children, `white-space: nowrap` on items that shouldn't compress, `padding-inline: clamp()` not fixed large values

## Pre-Ship Checklist

**Business layer:**
- [ ] What the site does and who it's for is clear within 5 seconds
- [ ] The primary CTA is specific, correctly weighted, and logically placed
- [ ] Objections are addressed before the conversion moment

**Design layer:**
- [ ] Palette + typographic voice + spatial energy couldn't belong to a different brand
- [ ] Every section earns its place — no filler between real content
- [ ] At least one design decision is unmistakably specific to this brand
- [ ] Visual hierarchy guides the eye without the user thinking about it

**UX layer:**
- [ ] Trust sequence is correct — proof before ask
- [ ] Information density is calibrated to the profile and audience
- [ ] The page has rhythm — dense and rest sections alternate
- [ ] Motion aids comprehension or creates appropriate feeling — doesn't decorate

**Copy layer:**
- [ ] Headlines couldn't work on a competitor's site
- [ ] Every CTA is specific to what the user actually wants to do
- [ ] No lorem ipsum, no "get started today", no template slogans

**Technical layer:**
- [ ] Mobile has intentional layout decisions, not just responsive stacking
- [ ] All fonts, images, and assets render without broken references
- [ ] If 3D Scroll Mode active: scroll state is legible, CTA remains findable, mobile has its own pacing design

## Output Standards

- Complete, runnable code. Not descriptions of what you'd build.
- Every section implemented. No "you could add X here."
- All assets inline or properly referenced — Google Fonts URLs, Unsplash photo IDs, inline SVGs.
- The output test: open the HTML file. Does it look like a finished site a real business could use today?
